### 二、1Panel 面板安装与常用命令

一键安装 1Panel，方便管理 Docker 与 Web 配置：

```bash
bash -c "$(curl -sSL https://resource.fit2cloud.com/1panel/package/v2/quick_start.sh)"
```

常用命令：

```
Usage:
  ./1pctl [COMMAND] [ARGS...]
  ./1pctl --help

Commands:
  status [core|agent]         检查 1Panel 服务状态
  start [core|agent|all]      启动 1Panel 服务
  stop [core|agent|all]       停止 1Panel 服务
  restart [core|agent|all]    重启 1Panel 服务
  uninstall                   卸载 1Panel 服务
  user-info                   获取 1Panel 用户信息
  listen-ip                   切换 1Panel 监听 IP
```

### Ech0-Noise 搭建笔记

#### Docker 原理图

```
+-------------------------------------------------+
|                  宿主机                        |
|                                                 |
| 访问地址: http://服务器IP:1314                  |
|                                                 |
|  +-----------------------------------------+   |
|  |             容器: Ech0-Noise           |   |
|  | 应用: Echo Noise                         |   |
|  | 内部端口: 1314                           |   |
|  | 数据存储卷: /app/data                     |   |
|  +-----------------------------------------+   |
| 本地挂载路径: /opt/data ------------------>     |
+-------------------------------------------------+
```

#### 常用命令与说明

##### 1️⃣ 启动容器（不挂载数据）

```bash
docker run -d   --name Ech0-Noise   -p 1314:1314   noise233/echo-noise
```

> ⚠️ 删除容器后数据会消失，适合测试环境。

##### 2️⃣ 数据持久化挂载

```bash
docker stop Ech0-Noise
docker rm -f Ech0-Noise
docker run -d   --name Ech0-Noise   -p 1314:1314   -v /opt/data/noise.db:/app/data/noise.db   noise233/echo-noise
```

将宿主机的 `/opt/data/noise.db` 挂载到容器 `/app/data/noise.db` 实现持久化。

##### 3️⃣ docker-compose 部署

```yaml
version: "3"

services:
  noise:
    image: noise233/echo-noise
    container_name: note-noise
    ports:
      - "1314:1314"
    volumes:
      - ./data:/app/data
    environment:
      - LOG_LEVEL=info
      - DB_TYPE=sqlite
      - DB_PATH=/app/data/noise.db
    restart: unless-stopped
```

一键启动：

```bash
docker compose up -d
```

#### `.env` 文件

```env
LOG_LEVEL=info
DB_TYPE=sqlite
DB_PATH=/app/data/noise.db
```

### Memos 搭建记录

Memos 是轻量级的自托管笔记系统。新版 UI 不太合我意，因此我选择回退旧版。

#### 部署命令

```yaml
version: "3.8"
services:
  memos:
    image: neosmemo/memos:0.24.0
    container_name: memos
    ports:
      - "5230:5230"
    volumes:
      - ./data:/var/opt/memos
    restart: unless-stopped
```

切换版本命令：

```bash
cd /opt/1panel/www/memos
docker compose down
docker stop memos
docker rm memos
docker compose up -d
```

### Nginx 反代注意事项

若访问反代域名时提示“无法访问”，说明浏览器可能强制使用 HTTPS（443 端口）。  
而你只配置了 HTTP（80 端口）。

**解决办法：**
1. 申请 HTTPS 证书（可用 Cloudflare 或 Let’sEncrypt 免费申请）。  
2. 若浏览器缓存了 HSTS，需清理缓存或使用隐身模式访问。

参考教程：  
[如何申请 Cloudflare 免费 SSL 证书](https://kerrynotes.com/apply-ssl-certificate-from-cloudflare/)
